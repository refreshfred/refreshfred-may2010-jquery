First: a word on Javascript ("JS")

  • Previously a "black sheep" of the web development world
  • Several different JS engines to target, all behave differently
  • Libraries (jQuery, Mootools, etc) exist to make your life easier

Enter: jQuery

  • Created by John Resig, Mozilla JS Evangelist
  • Open Source, shipping on 30% of all websites on the internet (CNN, The White House website, etc)
  • Backed in part by Microsoft, Nokia, and a slew of other companies.

What's jQuery good for?

  • DOM-abstraction layer, normalizes events and operations across all browsers
  • Short, simple CSS-based syntax for traversing the DOM (otherwise a complete pain)
  • Event handling and delegation (again, normalizing across all browsers)
  • Support for animating CSS properties of elements

Where jQuery isn't as strong

  • jQuery is a great library for DOM-work, but doesn't offer much outside of that scope
  • Pure JS additions don't come with jQuery - frameworks like Mootools offer a richer scenario around the computer science aspects (Classes, ECMAScript 5 functions, various other helpers)
  • Server-side work is almost non-existent with jQuery, just because it's so DOM based

Loading jQuery

  • jQuery is a bit of overhead - ~20kb on page load with gzipping, ~70kb without.
  • Rarely host it yourself - instead, pull it down from Google's CDN (Content Delivery Network)
  • http://ajax.googleapis.com/ajax/libs/jquery/1.4.2/jquery.min.js
  • Enough sites supply their users with this resource that your chances of having it cached are much higher than on your own.

Why jQuery Rocks: Sizzle, an awesome selector API

  • Sizzle is the open source, incredibly fast selector engine used in jQuery
  • jQuery selectors are based off of CSS selectors - reuse your existing knowledge
  • $("#container"); // Returns an object with a reference to a div id'd "container"
  • $(".slides"); // Returns an object with references to every slide
  • $("#container p"); // Returns an object with references to every <p> inside "container"

Check properties on those returned objects

  • $("#container").height(); // Returns a number
  • $("#container").width(); // Returns a number
  • $("#container").hasClass("lol"); // Boolean, returns true or false

$(document).ready(), and why it's important

  • Web browsers will parse and execute JS as it's fed to them
  • If you feed it too early, parts of the page might not exist yet
  • Enter: $(document).ready() - defers the work until the page is ready to go
  • 
<script type="text/javascript">
$(document).ready(function() {
    $("#container").html("<strong>Document's ready, so this'll actually work.</strong>");
});
</script>

Setting events with jQuery

  • jQuery further exposes the very "event-based" programming model of JS
  • $("#container").click(function() {
    alert("Woah, you clicked inside this container.");
});
  • $("#container").click(function() {
    // $(this) refers to the current object we're working on. Don't repeatedly lookup!
    $(this).fadeOut("fast");
});
  • $(".myInputs").keydown(function() {
	// .val() will return the value of an input or textarea
	alert("You typed: " + $(this).val()); 
});

Custom animations with jQuery

  • Animations in Javascript are based on CSS properties - position, width, height, etc
  • jQuery makes this incredibly easy, through use of .animate()
  • .animate() takes a property mapping, an optional animation speed, and an optional callback function
  • $("#container").click(function() {
    $(this).animate({width: $(this).width() + 100, height: $(this).height() + 100}, 
        800, 
	function() {
	        alert("Animation finished!");
        }
    );
});

The biggest misconception about jQuery

  • The biggest misconception is that jQuery makes it so you don't have to program.
  • You *are* still programming, so watch what you're doing.
  • The $() pattern in jQuery is a function, not a variable. Repeatedly calling it is *slow*. Cache things:
  • var container = $("#container");

for(var i = 0; i < 10; i++) { container.append("Tossing text inside"); }
  • is better than...
  • for(var i = 0; i < 10; i++) { $("#container").append("Tossing text inside"); }
